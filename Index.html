🛠 Environment Setup

Python ইনস্টল থেকে শুরু করে প্রথম Django প্রজেক্ট রান করা পর্যন্ত।

💡 টিপ: প্রতিটি প্রজেক্টের জন্য আলাদা Virtual Environment ব্যবহার করো। এতে প্যাকেজ কনফ্লিক্ট হবে না।
Virtual Environment তৈরি ও Activate BASH
python -m venv venv
👉 venv নামে একটি Virtual Environment তৈরি করে। এটি তোমার প্রজেক্টের জন্য আলাদা Python জগৎ।
Creates an isolated Python environment named "venv"
source venv/bin/activate
👉 Mac/Linux-এ venv চালু করো। Terminal-এ (venv) দেখা যাবে।
Activates venv on Mac/Linux — you'll see (venv) in your prompt
venv\Scripts\activate
👉 Windows-এ venv চালু করার কমান্ড।
Activates venv on Windows
deactivate
👉 venv বন্ধ করো। আবার global Python-এ ফিরে যাবে।
Deactivates the virtual environment
Django ইনস্টল ও প্রজেক্ট শুরু BASH
pip install django
👉 সর্বশেষ Django ইনস্টল করো। Install latest Django
pip install django==4.2
👉 নির্দিষ্ট ভার্সন ইনস্টল করো। LTS ভার্সন (4.2) প্রডাকশনে ভালো। Install specific version — 4.2 is LTS
django-admin startproject myproject .
👉 নতুন Django প্রজেক্ট তৈরি করো। শেষে . দিলে current folder-এই তৈরি হবে। Creates a new Django project in current directory
python manage.py runserver
👉 Development server চালু করো। Browser-এ http://127.0.0.1:8000 খোলো। Starts dev server at localhost:8000
python manage.py runserver 0.0.0.0:8080
👉 নির্দিষ্ট port-এ সার্ভার চালাও। Network-এর অন্য device থেকেও access করা যাবে। Run on specific port, accessible on local network

🗄 Models & Database

Django ORM দিয়ে database ডিজাইন, query ও optimization।

blog/models.py — সম্পূর্ণ Model উদাহরণ PYTHON 
from django.db import models
from django.contrib.auth.models import User

# ── Category Model ──────────────────────────
class Category(models.Model):
    name = models.CharField(max_length=100, unique=True)
    slug = models.SlugField(unique=True)  # URL-friendly নাম
    
    class Meta:
        verbose_name_plural = "categories"
        ordering = ['name']  # নাম অনুযায়ী sort
    
    def __str__(self):
        return self.name

# ── Post Model ──────────────────────────────
class Post(models.Model):
    # Status choices — tuple-এর list
    STATUS_CHOICES = [
        ('draft', 'Draft'),
        ('published', 'Published'),
    ]
    
    title       = models.CharField(max_length=200)
    slug        = models.SlugField(unique=True)
    content     = models.TextField()
    excerpt     = models.TextField(blank=True)  # Optional field
    image       = models.ImageField(upload_to='posts/', null=True, blank=True)
    status      = models.CharField(max_length=10, choices=STATUS_CHOICES, default='draft')
    views_count = models.PositiveIntegerField(default=0)
    
    # Timestamps — auto_now_add=True মানে create-এর সময় set হয়
    created_at  = models.DateTimeField(auto_now_add=True)
    updated_at  = models.DateTimeField(auto_now=True)  # save-এর সময় update হয়
    
    # Relationships
    author      = models.ForeignKey(User, on_delete=models.CASCADE, related_name='posts')
    category    = models.ForeignKey(Category, on_delete=models.SET_NULL, null=True)
    tags        = models.ManyToManyField('Tag', blank=True)  # Many-to-Many
    
    class Meta:
        ordering = ['-created_at']  # নতুন পোস্ট আগে
        indexes = [
            models.Index(fields=['status', '-created_at']),  # ⚡ Query fast করে
        ]
    
    def __str__(self):
        return self.title
    
    def get_absolute_url(self):  # canonical URL
        from django.urls import reverse
        return reverse('post-detail', kwargs={'slug': self.slug})
ORM Queries — Shell-এ চেষ্টা করো PYTHON
Post.objects.all()
👉 সব Post পাও। QuerySet return করে — এখনো DB hit করে না (lazy)।
Post.objects.filter(status='published')
👉 শুধু published পোস্ট ফিল্টার করো।
Post.objects.filter(title__icontains='django')
👉 Title-এ 'django' আছে এমন পোস্ট (case-insensitive)। __icontains = field lookup।
Post.objects.exclude(status='draft')
👉 draft বাদ দিয়ে সব পোস্ট পাও।
Post.objects.get(slug='my-post')
👉 একটি নির্দিষ্ট পোস্ট পাও। না পেলে DoesNotExist error। একাধিক পেলে MultipleObjectsReturned।
Post.objects.get_or_create(slug='test', defaults={'title': 'Test'})
👉 থাকলে পাও, না থাকলে তৈরি করো। Returns (object, created_bool)।
Post.objects.select_related('author', 'category')
👉 ⚡ ForeignKey relationship-এর data একটি query-তেই পাও (JOIN করে)। N+1 সমস্যা দূর করে।
Post.objects.prefetch_related('tags')
👉 ⚡ ManyToMany relationship-এর জন্য। আলাদা query করে cache-এ রাখে।
Post.objects.order_by('-created_at', 'title')
👉 নতুন আগে, তারপর title অনুযায়ী sort করো।
Post.objects.count()
👉 মোট পোস্ট সংখ্যা। DB-তে COUNT() চালায়।
Post.objects.values('title', 'status')
👉 শুধু নির্দিষ্ট field-এর dict list পাও। পুরো object না পেলে দ্রুত।
Post.objects.aggregate(total=Count('id'), avg=Avg('views_count'))
👉 পুরো QuerySet-এর aggregate calculation। from django.db.models import Count, Avg দরকার।
Post.objects.annotate(comment_count=Count('comments'))
👉 প্রতিটি পোস্টে মন্তব্যের সংখ্যা যোগ করে একটি extra field হিসেবে।
⚠️ সাবধান: Post.objects.all() সব data memory-তে load করে। বড় dataset-এ সবসময় filter(), .only('field1') বা pagination ব্যবহার করো।

blog/views.py — Function-Based Views (FBV) PYTHON 
from django.shortcuts import render, get_object_or_404, redirect
from django.contrib.auth.decorators import login_required
from django.views.decorators.http import require_http_methods
from django.http import JsonResponse
from .models import Post

# ── সাধারণ List View ────────────────────────
def post_list(request):
    posts = Post.objects.filter(status='published').select_related('author')
    return render(request, 'blog/list.html', {'posts': posts})

# ── Login দরকার এমন View ────────────────────
@login_required  # না থাকলে login page-এ redirect করবে
def post_create(request):
    if request.method == 'POST':
        title   = request.POST.get('title')
        content = request.POST.get('content')
        post    = Post.objects.create(title=title, content=content, author=request.user)
        return redirect('post-detail', slug=post.slug)
    return render(request, 'blog/create.html')

# ── JSON Response (API-like) ─────────────────
@require_http_methods(["GET"])
def post_json(request, pk):
    post = get_object_or_404(Post, pk=pk)  # না পেলে 404 দেয়
    return JsonResponse({'title': post.title, 'status': post.status})
blog/views.py — Class-Based Views (CBV) — বেশি শক্তিশালী PYTHON 
from django.views.generic import ListView, DetailView, CreateView, UpdateView, DeleteView
from django.contrib.auth.mixins import LoginRequiredMixin
from django.urls import reverse_lazy

# ── ListView: সব পোস্টের লিস্ট ──────────────
class PostListView(ListView):
    model        = Post
    template_name = 'blog/list.html'
    context_object_name = 'posts'  # template-এ 'posts' নামে পাবে
    paginate_by  = 10  # এক পাতায় ১০টি — pagination auto!
    
    def get_queryset(self):  # Custom filter
        return Post.objects.filter(status='published')

# ── DetailView: একটি পোস্ট ────────────────────
class PostDetailView(DetailView):
    model        = Post
    template_name = 'blog/detail.html'
    slug_field   = 'slug'  # URL-এ slug ব্যবহার করা হবে

# ── CreateView: নতুন পোস্ট ────────────────────
class PostCreateView(LoginRequiredMixin, CreateView):
    model         = Post
    fields        = ['title', 'content', 'category']
    template_name = 'blog/form.html'
    success_url   = reverse_lazy('post-list')  # সফল হলে কোথায় যাবে
    
    def form_valid(self, form):
        form.instance.author = self.request.user  # author auto-set
        return super().form_valid(form)

# ── DeleteView ────────────────────────────────
class PostDeleteView(LoginRequiredMixin, DeleteView):
    model       = Post
    success_url = reverse_lazy('post-list')

blog/serializers.py — Data Validation & Transformation PYTHON 
from rest_framework import serializers
from .models import Post, Category

class CategorySerializer(serializers.ModelSerializer):
    class Meta:
        model  = Category
        fields = ['id', 'name', 'slug']

class PostSerializer(serializers.ModelSerializer):
    # Nested serializer — category-র পুরো object দেখাবে, শুধু id না
    category = CategorySerializer(read_only=True)
    # Write করতে id দেওয়ার জন্য আলাদা field
    category_id = serializers.IntegerField(write_only=True)
    # Custom field — model-এ নেই কিন্তু response-এ থাকবে
    author_name = serializers.SerializerMethodField()
    
    class Meta:
        model  = Post
        fields = ['id', 'title', 'slug', 'content', 'status',
                  'category', 'category_id', 'author_name', 'created_at']
        read_only_fields = ['id', 'slug', 'created_at']
    
    def get_author_name(self, obj):  # SerializerMethodField-এর method
        return obj.author.get_full_name() or obj.author.username
    
    def validate_title(self, value):  # Field-level validation
        if len(value) < 5:
            raise serializers.ValidationError("Title must be at least 5 characters.")
        return value
blog/views.py — ViewSets — সবচেয়ে কম কোডে পুরো CRUD PYTHON 
from rest_framework import viewsets, permissions, filters
from rest_framework.decorators import action
from rest_framework.response import Response
from django_filters.rest_framework import DjangoFilterBackend
from .models import Post
from .serializers import PostSerializer

class PostViewSet(viewsets.ModelViewSet):
    """
    GET  /api/posts/         → list
    POST /api/posts/         → create
    GET  /api/posts/1/       → retrieve
    PUT  /api/posts/1/       → update
    DEL  /api/posts/1/       → destroy
    """
    queryset           = Post.objects.select_related('author', 'category')
    serializer_class   = PostSerializer
    permission_classes = [permissions.IsAuthenticatedOrReadOnly]
    
    # Filtering, Searching, Ordering
    filter_backends  = [DjangoFilterBackend, filters.SearchFilter, filters.OrderingFilter]
    filterset_fields = ['status', 'category']      # ?status=published
    search_fields    = ['title', 'content']        # ?search=django
    ordering_fields  = ['created_at', 'views_count'] # ?ordering=-views_count
    
    def perform_create(self, serializer):
        serializer.save(author=self.request.user)  # Author auto-set
    
    # Custom action — GET/POST /api/posts/trending/
    @action(detail=False, methods=['get'])
    def trending(self, request):
        trending = self.get_queryset().order_by('-views_count')[:5]
        serializer = self.get_serializer(trending, many=True)
        return Response(serializer.data)

Environment Variables সেট করো

SECRET_KEY, DATABASE_URL, DEBUG=False সব .env ফাইলে রাখো। কখনো code-এ লিখবে না।

2

Gunicorn দিয়ে WSGI Server চালাও

pip install gunicorn তারপর gunicorn config.wsgi:application --workers 3

3

Static Files Collect করো

python manage.py collectstatic --noinput — Nginx দিয়ে static serve করবে।

4

Nginx Reverse Proxy

Nginx → Gunicorn → Django। HTTPS SSL, static file serving Nginx করবে।

5

Database Migration Production-এ

python manage.py migrate --run-syncdb — তারপর createsuperuser।

Docker দিয়ে Deploy BASH
docker build -t myapp .
👉 Docker image তৈরি করো।
docker-compose up -d
👉 Django + PostgreSQL + Nginx সব একসাথে background-এ চালাও। docker-compose.yml দরকার।
docker-compose exec web python manage.py migrate
👉 চলমান container-এ manage.py command চালাও।
docker-compose logs -f web
👉 Real-time log দেখো। Error debugging-এ কাজে লাগে।
💡 Production Checklist: python manage.py check --deploy চালাও — Django নিজেই security issues ধরিয়ে দেবে।

🗺 Basic → Pro Roadmap

ধাপে ধাপে Django expert হওয়ার পথ।

🟢 Stage 1 — Foundation Basic
  • Python OOP ভালো করে শেখো (class, inheritance)
  • Django MVT concept বোঝো
  • Virtual environment, pip
  • First app তৈরি (Blog/Todo)
  • Models, Views, Templates, URLs
  • Django Admin customization
  • Forms ও Validation
🔵 Stage 2 — Intermediate Mid
  • Class-Based Views (ListView, DetailView, CRUD)
  • Django ORM advanced (annotate, aggregate, Q objects)
  • User Authentication (login, logout, register)
  • File Upload ও Media handling
  • Django REST Framework (Serializers, APIView)
  • JWT Token Authentication
  • Signals (post_save, pre_delete)
  • Custom Management Commands
🟠 Stage 3 — Advanced Advanced
  • Query Optimization (select_related, prefetch, indexes)
  • Caching (Redis, Memcached)
  • Celery Task Queue (async tasks)
  • WebSocket (Django Channels)
  • Custom Middleware
  • Testing (pytest, factory_boy, mock)
  • API Versioning ও Swagger docs
  • Docker + CI/CD pipeline
🔴 Stage 4 — Pro Expert
  • Microservices architecture
  • Database Sharding ও Replication
  • Custom Authentication backends
  • Multi-tenancy (django-tenants)
  • Performance profiling (django-debug-toolbar, silk)
  • Search (Elasticsearch, django-haystack)
  • Cloud deploy (AWS, GCP, DigitalOcean)
  • Kubernetes orchestration
